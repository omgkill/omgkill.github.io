ams2 压测服game服务容器化

  1. 由于压测都是找空闲的机器来测试，所以每次压测都要重新部署环境，为了简化流程，想通过docker简化操作

  2. 参考lost私服做的修改

    • lost私服:http://confluence.super-chameleon.com:8093/pages/viewpage.action?pageId=75476077

  3. 文件路径

    • http://svn.super-chameleon.com:8822/svn/AMS_TD/Source/trunk/Server/Tools/Ops/ds

  4. 使用jenkins构建docker镜像

    • jenkins地址：http://10.0.3.2:7005/

  5. 使用nexus3作为docker镜像仓库

    • 仓库地址：http://10.0.3.2:8082/

  6. docker 管理界面

    • 地址：http://10.0.3.2:9000/

  7. 关于base的构建

    • base的意思是搭建一个基础环境

    • 包含cetos7、jdk安装、mysql安装、redis安装、ssh安装

    • jenkins构建base

      • 进入jenkins界面，点击build_ams2_base

        ​

      • 再点击立即构建，就会自动编译，并编译推送到仓库

    • jenkins配置：http://svn.super-chameleon.com:8822/svn/AMS_TD/Source/trunk/Server/Tools/Ops/ds/jenkins/build_ams2_base.jenkins

    • dockerfile以及其他文件：http://svn.super-chameleon.com:8822/svn/AMS_TD/Source/trunk/Server/Tools/Ops/ds/docker/base

  8. 关于ams2的构建

    • 这个就是真正构建game服务
    • 包含redis启动、mysql启动、mysql数据导入、game部署文件、game启动
    • jenkins构建base
      • 进入jenkins界面，点击build_ams2_ds
      • 再点击立即构建，就会自动编译，并编译推送到仓库
      • jenkins配置：http://svn.super-chameleon.com:8822/svn/AMS_TD/Source/trunk/Server/Tools/Ops/ds/jenkins/build_ams2_ds.jenkins
      • dockerfile以及其他文件：http://svn.super-chameleon.com:8822/svn/AMS_TD/Source/trunk/Server/Tools/Ops/ds/docker/ams2

  9. 说一下build_ams2_base 这个任务如何创建的。build_ams2_ds同理

    • 点击新建任务，
    • 输入任务名字，点击流水线，再点击报错
    • 填写描述，点击不允许并发构建
    • 按照如下配置，Credentials是svn的账号密码
    • 还有一个脚本路径，这个是相对路径。相对于svn down的目录
    • 点击保存就好了
