使用Docker启动压测服

by M
2023-01-01
(LastMod: 2023-03-07)
1354 words - 3 min read
docker
  1. gm服部署nexus3，这个用于docker image 仓库。
    • 参考：http://confluence.super-chameleon.com:8093/pages/viewpage.action?pageId=75479441
  2. gm服部署jenkins
    • 参考：http://confluence.super-chameleon.com:8093/pages/viewpage.action?pageId=75479445
    • 需要安装docker plugin
      • 插件管理中安装Docker, 系统管理->插件管理-> 可选插件->搜索docker
      • 配置docker镜像仓库地址和密钥，路径：系统管理->系统配置-> Declarative Pipeline (Docker)
    • 安装gradle plugin
      • 配置gradle，名字：gradle564，版本：Gradle 5.6.4，路径：系统管理->全局工具配置-> Gradle
  3. jenkins配置
    • 参考：http://confluence.super-chameleon.com:8093/pages/viewpage.action?pageId=96209340
  4. 构建ams2-base 和 ams2-ds
  5. svn checkout 脚本
    • svn地址：http://svn.super-chameleon.com:8822/svn/AMS_TD/Source/trunk/Server/Tools/Ops/ds/deploy_stressing
  6. 进入deploy_stressing文件夹
  7. 执行脚本， 默认启动8001服
    • python deploy_stressing_server.py -g <game服内网ip> -d <mysql与clusterRedis服内网ip> -n <docker镜像仓库所在服内网ip>
  8. 如果要重启game服，要多加一个参数：-r
    • python deploy_stressing_server.py -g <game服内网ip> -d <mysql与clusterRedis服内网ip> -n <docker镜像仓库所在服内网ip> -r true
  9. 脚本支持重复执行，如果脚本中途执行失败，可以重复执行
  10. docker储存到宿主机的数据
    • mysql的数据存储地址，在宿主机的:/home/docker/docker/volume/stress_db_mysql
    • redis的数据存储地址，在宿主机：/home/docker/docker/volume/stress_db_redis
    • game log的数据存储地址，在宿主机：/home/docker/docker/volume/stress_game_log
  11. 示例
    • python .\deploy_stressing_server.py -g 10.0.3.203 -d 10.0.3.21 -n 10.0.3.2

##脚本说明

  1. 脚本需要两台服务器。 一个台部署game与local redis，一台部署mysql与cluster redis
  2. 两台机器容器如何互联，参考：http://confluence.super-chameleon.com:8093/pages/viewpage.action?pageId=96209514
  3. 如果要删除所有docker数据，执行一下命令
    • yum remove -y docker*
